# 设计：单标签页内导航（消除标签页累积）

- 日期：2026-06-29
- 脚本：`jd-auto-review.user.js`（当前 `@version 8.5` → 目标 `8.6`）
- 状态：已通过 brainstorming，待用户审阅后进入实现

## 背景 / 问题

用户反馈：挂机批量评价（京东评价常按月集中处理几十单）时，脚本每完成一单、进入下一单就会新开一个浏览器标签页，跑完几十单后标签页累积到几十个，配置一般的电脑容易爆内存。

## 目标

- 让整条闭环（列表 → 评价 → 成功 → 列表）**始终在同一个标签页内**完成，从源头不再新增标签页。
- 改动集中、最小，不破坏现有「开始 / 暂停」状态机与暂停边界语义。

## 非目标

- 不自动回收**已经**在历史运行中堆出来的旧标签页（浏览器禁止脚本关闭非脚本打开的标签页，技术上不可行；用户需手动关一次）。
- 不改变评价生成、配图、打星等非导航逻辑。
- 不新增配置开关（始终开启单标签导航）。

## 根因

脚本的三个跳转步骤全部经 `autoClickAfter()`（`jd-auto-review.user.js:141`）执行 `$el[0].click()`：

| 步骤 | 函数 | 动作 |
|------|------|------|
| 列表 → 评价 | `runListStep` | 点 `.operate` 下「评价」 |
| 评价 → 成功 | `publishStep` | 点文案「发表」 |
| 成功 → 列表 | `successReturnStep` | 点「返回待评价列表」 |

京东这三个链接几乎都是 `<a target="_blank">`，而 `autoClickAfter` 点击前**完全没有处理 `target`**，于是每次跳转新开一个标签页、旧标签页留着不关 → 累积。

用户反馈「几乎每一步都会新开」，印证三个跳转点都是 `target="_blank"`，因此修法可集中在 `autoClickAfter` 这一处收口。

## 方案选择

- **方案 A（采用）：去 `target="_blank"` + 临时接管 `window.open` 兜底。** 点击前把目标元素及祖先 `<a>`/`<form>` 的 `target` 强制设 `_self`；并在点击那一瞬临时把 `window.open` 改为原地 `location.href` 跳转，点完立即还原。一处改动覆盖三个跳转点，且能兜住京东用 onclick 的 `window.open(...)` 程序化开新页的情况。
- **方案 B（未采用）：只去 `target`，不接管 `window.open`。** 代码最少，但若某步用程序化 `window.open` 开新页则兜不住。无法对用户实时京东页面联调，单独用 B 有风险。
- **方案 C（已评估、放弃）：用 `window.opener` 关上一个标签页。** 即字面意义的「自动关闭多余标签页」。但浏览器禁止脚本关闭非脚本打开的标签页，`window.close()` 会被拦截或弹确认框，不可靠。

## 设计详情

### 改动位置

**仅** `autoClickAfter`（`jd-auto-review.user.js:141`）。打星 `.star5`、回填 textarea、调 LLM 等非导航动作不动。

### 新增工具函数

1. **`forceSameTabNav($el)`**：从被点元素向上遍历祖先链，对遇到的 `<a>` 或 `<form>`，将其 `target` 设为 `_self`（并移除原 `target="_blank"` 等取值）。覆盖「元素本身即 `<a>`」与「元素是 `<a>` 内的 `<span>`」两种形态。

2. **`withOpenGuard(fn)`**：
   - 执行 `fn` 前，临时把 `window.open` 替换为：`(url) => { if (url) location.href = url; return null; }`。
   - 在 `finally` 中还原原始 `window.open`；另设一个约 500ms 的保险计时器再次还原，兜住京东用 `setTimeout` 延迟 `window.open` 的情况。
   - **还原必须执行**：若某次点击未触发跳转（如发表表单校验未过留在原地），不能让 `window.open` 永久失效。

### 改写点击块

`autoClickAfter` 内，点击部分由：

```js
$el[0].click();
```

改为：

```js
forceSameTabNav($el);
withOpenGuard(function(){ $el[0].click(); });
```

（外层包 `try/catch`，中和异常时退回普通 `click()`。）

### 优雅降级

- 整段包 `try/catch`：中和失败 → 退回普通 `click()` → 最坏情况等同现状（开新标签页），**不会比现在更差**。
- `withOpenGuard` 的还原走 `finally` + 保险计时器双保险，必执行。
- 暂停 / 续作语义零改动：单标签页内导航照样携带 `localStorage` 的 `running` 标记，与脚本原设计的单标签循环一致。

### 配置

不新增开关（YAGNI）。始终单标签导航。

## 测试

唯一可行方式为浏览器手测（无构建 / 无自动化测试框架）：

1. 在列表页点「开始」，盯住浏览器标签数。
2. 完整跑一轮 `列表 → 评价 → 成功 → 列表`，再连续多轮，标签数应**始终为 1**。
3. 在三个页面（列表 / 评价 / 成功）分别确认跳转均在当前标签页内完成。
4. 边界确认：历史运行已堆出的旧标签页**不会**被自动关闭（符合预期，需手动关一次）。
5. 兜底回归：暂停 / 续作仍停在步骤边界，`running` 跨页保持。

## 风险

- 若京东未来把跳转改为既非 `target="_blank"`、又非 `window.open` 的方式（如 `location.assign` 之外的某种机制），`window.open` 兜底可能漏；此时标签数会上升，可借手测第一轮观察发现。属可接受风险，不影响主路径。
- `window.open` 临时接管窗口期内若有其它合法 `window.open` 调用会被改写为原地跳转；由于窗口仅覆盖一次点击且导航后页面上下文即销毁，影响可忽略。
